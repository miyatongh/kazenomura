# 2026/2/16 Phase 0「要求設計」コンサルティング説明会
# スライド構成案 v3.1（思考過程ベース・管理階層起点）

## 構成の基本方針

**従来の構成（既存案）：**
```
方法論の説明 → 活動の説明 → 成果物の説明 → お願い
（何をやるか）   （どう進めるか）（何ができるか）（何をしてほしいか）
```

**今回の構成：**
```
思考の出発点 → 思考の展開 → 思考が導く結論 → 一緒に進める理由
（なぜそう考えるか）→（考えるとこう見える）→（だからこうなる）→（だから一緒にやる）
```

**なぜこの構造にするか：**
通常のコンサルティングでは「○○をやります、結果はこうでした」しか見せない。
しかし本質は結果ではなく、結果を導く思考過程にある。
思考の仕方が違えば、結果は自ずと違う。
思考そのものが理解されていなければ、コンサルティングの本来の意味をなさない。
逆に、思考そのものに賛同いただければ、仮によくない結果が出たとしても、
思考そのものは生き続ける。

**対象：** 理事長はじめ幹部15名（非IT）、Web会議、1時間（説明38分＋質疑22分）

## C. 思考の核心：業務には「本来の構造」がある（スライド7-14）── 15分

**狙い：** ここがこの説明会の最重要パート。
まず「管理階層」という組織の骨格を示す。
各階層は管理対象のライフサイクル（生まれてから亡くなるまで）を管理しており、
**そもそも全部つながっている**。一気通貫は「目標」ではなく「本来の姿」。
この構造の上で、仕事がどう動くか（Customer-Performer、PD管理、一個流し）を示す。
そして、風の村の課題が、この本来の構造からの逸脱として説明できることを示す。

**v3からの最大の変更点：**
v3では3原則を並列に提示した。v3.1では管理階層を土台に据え、
3原則はその構造から自然に導かれるものとして順序立てる。
特に「一気通貫」は原則の一つではなく、管理階層の帰結として先に示す。

## 構成の設計意図（メモ）

### 既存案との最大の違い

既存の構成案（33枚版・27枚版とも）は、4テーマを並列に並べる構造だった：
①考え方 ②活動 ③成果物 ④お願い

今回の構成は、**一本の思考の筋道**を追体験する構造にした：
- B「全体を見る」は、Cの前提になる
- C「本来の構造がある」は、Dの前提になる
- D「だからこう進める」は、Cの論理的帰結である
- E「一緒にやる」は、D-19の「機能は皆さんが発明する」の帰結である

**一つ前の話が、次の話の「なぜ」になっている。**
この連鎖が途切れなければ、聴き手は「なるほど、だからそうなるのか」と感じる。

### v3 → v3.1 の変更点（管理階層の位置づけ）

v3ではCセクションで3原則（一気通貫・Customer-Performer・PD管理＋一個流し）を並列に提示した。
v3.1では**管理階層を土台に据え**、そこから3原則が自然に導かれる構造に変更した：

```
管理階層（3つの管理レベル）
  └→ 各レベルはライフサイクル全体を管理している
      └→ だからそもそも一気通貫している（帰結であり、目標ではない）
          └→ つながった仕事は「依頼→約束→実行→検収」で動く
              └→ 計画と実績はセット、完結はその場で
```

この変更により：
- 「一気通貫」が原則の一つから、管理階層の**論理的帰結**に昇格した
- 聴き手は「なぜつながっているのか」の理由を先に理解できる
- 風の村の課題が「本来つながっているものが切れた結果」として明確に位置づく

### スライドデザイン方針
- 1スライド最大5-6行、フォント18pt以上（Web会議視認性）
- IT用語は使わない：EA→「4つのレンズ」、Customer-Performer→「依頼→約束→実行→検収」
- 比喩の統一：都市計画、レストランの注文、健康診断
- セクション間にブリッジとなる一文を必ず入れる（"だから""この考え方に基づいて"）
- スライド13（課題の読み解き）は、左右対比で視覚的に構造を見せる

### 除外事項
- 育成研修（全15回）：本編には入れない。質疑で聞かれたら口頭で補足
- 見積金額：スライドには含めない
- 技術的詳細（JIS規格番号、M-GTA等の手法名）：スライドには出さない

### 23枚にした理由
38分で23枚 ＝ 1枚あたり約1.7分。Web会議で非IT幹部向けには適正なペース。
既存案の33枚は1枚1.2分で、Web会議では速すぎるリスクがあった。
Cセクションを8枚（15分）に拡充したのは、管理階層→一気通貫の論理展開に
十分な時間が必要なため。ここが説明会の核心であり、時間配分も最大とした。
